# Env

<Intro>
The env command prints a formatted list of environment variables defined in a blueprint which can be interpreted by the operating system.
</Intro>

## Usage

```shell
jumppad env [flags]
```

This can be very useful to either set or unset environment variables on your current session.

<CodeGroup title="Set environment variables on your current session.">
```bash {{ title: "Linux" }}
  eval $(jumppad env)
```

```powershell {{ title: "Windows" }}
Invoke-Expression "jumppad env" | ForEach-Object { Invoke-Expression $_ }
```
</CodeGroup>

<CodeGroup title="Unset environment variables on your current session.">
```bash {{ title: "Linux" }}
eval $(jumppad env --unset)
```

```powershell {{ title: "Windows" }}
Invoke-Expression "jumppad env --unset" | ForEach-Object { Remove-Item $_ }
```
</CodeGroup>

## Flags

<Properties>
  <Property name="unset" type="boolean" required="false" value="false">
    When set to true Jumppad will print unset commands for environment variables defined by the blueprint.
  </Property>
</Properties>
